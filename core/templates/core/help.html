¿Qué es Simona DataPipeline?

Simona DataPipeline es una plataforma que te permite sincronizar automáticamente datos desde bases de datos PostgreSQL hacia servicios de almacenamiento en la nube (Google Drive y Amazon S3) y gestionar la limpieza automática de datos antiguos basándose en políticas de retención personalizadas.

Google Drive Amazon S3 Limpiezas Automáticas

Conectores Google Drive

¿Cómo crear un conector?

  1. Haz clic en "Mis Conectores Google Drive" en el menú
  2. Presiona el botón "Crear Nuevo Conector"
  3. Completa los datos de conexión PostgreSQL (host, puerto, base de datos, usuario, contraseña)
  4. Elige la frecuencia de sincronización automática
  5. Presiona "Configurar Google Drive" para autorizar el acceso
  6. Selecciona las tablas que deseas sincronizar

Nota: La primera vez necesitarás autenticar tu cuenta de Google para permitir que Simona suba archivos a tu Drive.

Conectores Amazon S3

¿Cómo crear un conector?

  1. Haz clic en "Mis Conectores Amazon S3" en el menú
  2. Presiona el botón "Crear Nuevo Conector"
  3. Completa los datos de conexión PostgreSQL
  4. Ingresa tus credenciales de AWS (Access Key, Secret Key, Región)
  5. Especifica el nombre del bucket S3 donde se guardarán los archivos
  6. Elige la frecuencia de sincronización
  7. Selecciona las tablas que deseas sincronizar

Nota: Asegúrate de tener permisos de escritura en el bucket S3 especificado.

Tareas de Sincronización

Cada tabla que sincronizas tiene su propia tarea independiente. Las tareas de sincronización se ejecutan automáticamente según la frecuencia configurada en el conector.

Funcionamiento de la sincronización:

  • Primera sincronización: Exporta todos los datos de la tabla a formato CSV
  • Sincronizaciones siguientes: Solo exporta las filas nuevas basándose en la columna timestamp que seleccionaste
  • Cambios de esquema: Si la estructura de la tabla cambia, se crea un nuevo archivo con el esquema actualizado
  • Archivos viejos: Se renombran con "_OLD_[fecha]" cuando hay cambios de estructura

Ejecución manual:

Puedes ejecutar una sincronización inmediatamente usando el botón "Sincronizar Ahora" en lugar de esperar la programación automática.

Importante: Para que la sincronización incremental funcione correctamente, cada vez que actualices registros en tu base de datos, debes actualizar el campo timestamp (por ejemplo: updated_at = NOW()). Esto permite que Simona detecte qué registros son nuevos o fueron modificados desde la última sincronización.

Limpiezas Automáticas

Las limpiezas automáticas te permiten eliminar datos antiguos de tus bases de datos PostgreSQL basándose en políticas de retención.

¿Cómo crear una limpieza automática?

  1. Haz clic en "Mis Limpiezas Automáticas" en el menú
  2. Presiona "Crear Nueva Limpieza"
  3. Asigna un nombre descriptivo
  4. Completa los datos de conexión PostgreSQL
  5. Ingresa el nombre de la tabla a limpiar
  6. Presiona "Cargar Campos Timestamp" para ver las columnas de fecha disponibles
  7. Selecciona la columna timestamp que define la antigüedad de los datos
  8. Configura la retención (meses, días, horas)
  9. Elige la frecuencia de ejecución (1, 6, 12 o 24 horas)

Política de retención:

La retención se calcula como: meses × 30 días + días + horas. Los datos más antiguos que este período serán eliminados automáticamente.

Advertencia: La eliminación de datos es irreversible. Asegúrate de tener respaldos antes de configurar limpiezas automáticas.
Permisos requeridos: Asegúrate de que el usuario de PostgreSQL tenga los permisos necesarios: 
GRANT SELECT, DELETE ON TABLE nombre_tabla TO simona_user;

Historial de ejecuciones:

Cada limpieza guarda un historial con las últimas 20 ejecuciones, mostrando cuántas filas fueron eliminadas, el tiempo de duración y el estado (exitoso o fallido).

Configuración de Cuenta

En la sección de Configuración puedes actualizar tu información personal:

  • Nombre de usuario: Debe ser único en el sistema
  • Email: Tu dirección de correo electrónico
  • Contraseña: Para cambiarla, necesitas ingresar tu contraseña actual y confirmar la nueva (mínimo 8 caracteres)

Preguntas Frecuentes

¿Puedo pausar un conector temporalmente?

Sí, puedes desactivar un conector editándolo y desmarcando la opción "Activo". Esto detendrá las sincronizaciones automáticas sin eliminar la configuración ni el historial.

¿Qué pasa si cambia la estructura de mi tabla?

Simona detecta cambios de esquema automáticamente. Cuando detecta un cambio, crea un nuevo archivo con el esquema actualizado y renombra el archivo anterior agregando "_OLD_" seguido de la fecha.

¿Puedo sincronizar la misma tabla a Google Drive y S3?

Sí, puedes crear dos conectores diferentes (uno para Drive y otro para S3) apuntando a la misma base de datos y seleccionar las mismas tablas en ambos.

¿Cómo sé si una limpieza automática está funcionando?

Puedes ver el historial de ejecuciones en la página de detalle de cada limpieza. Allí verás cuándo se ejecutó, cuántas filas fueron eliminadas y si hubo algún error.

¿Qué formato tienen los archivos sincronizados?

Todos los datos se exportan en formato CSV (valores separados por comas) con codificación UTF-8, compatible con Excel, Google Sheets y otras herramientas de análisis.
